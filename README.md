> 🚀 2026 优化版 | 适配 Android 14/15 | AndroidX + OkHttp 4 + ZXing 3.5

## 📖 这是什么？

V免签是一款 **免费开源** 的个人收款监控工具。它通过 Android 手机监听微信和支付宝的 **收款通知**，自动将收款信息推送到你自己的服务器，实现无需签约支付平台的个人收款回调方案。

**简单来说：** 手机收到微信/支付宝到账通知 → App 自动识别金额 → 推送到你的服务器 → 你的业务系统收到回调。

### 适合谁用？

- ✅ 个人开发者，需要收款回调功能
- ✅ 小型站点，无法接入官方支付 API
- ❌ 不适合商用多用户场景
- ❌ 不适合零技术基础用户（需要部署服务端）

## ⚡ 5 分钟快速上手

### 第一步：部署服务端

选择一个服务端版本，部署到你的服务器：

| 版本 | 地址 | 说明 |
|------|------|------|
| Java 版 | [github.com/szvone/Vmq](https://github.com/szvone/Vmq) | SpringBoot，推荐 |
| PHP 版 | [github.com/szvone/vmqphp](https://github.com/szvone/vmqphp) | ThinkPHP，需配置伪静态 |

部署完成后，登录后台 → 修改系统设置 → 打开 **监控端设置页面**，你会看到一个二维码和配置信息。

### 第二步：安装 Android 监控端

**方式 A：直接安装 APK（推荐）**

1. 下载 `app/app-release.apk`
2. 传输到 Android 手机，点击安装
3. 如提示"未知来源"，请在设置中允许安装

**方式 B：自行编译**

```bash
git clone https://github.com/你的仓库地址/vmqApk.git
cd vmqApk
./gradlew assembleDebug
# 产物在 app/build/outputs/apk/debug/app-debug.apk
```

> ⚠️ 要求：Android SDK 35、JDK 17+

### 第三步：配置监控端

打开 V免签 App 后，按照以下步骤操作：

```
1️⃣ 授权通知使用权
   App 会自动跳转到系统设置页面
   找到 "V免签" → 开启通知使用权
   ↓
2️⃣ 配置服务端连接
   方式1: 点击 [扫码配置] → 扫描服务端后台的二维码（推荐✅）
   方式2: 点击 [手动配置] → 输入 "服务器地址/密钥"
   ↓
3️⃣ 测试连接
   点击 [检测心跳] → 应提示 "心跳返回：success" 或类似成功信息
   ↓
4️⃣ 测试监听
   点击 [检测监听] → 应弹出 Toast 提示 "监听正常！通知捕获成功"
   ↓
5️⃣ 完成！🎉
```

### 第四步：确保后台存活（重要！）

为防止系统杀死 App，**必须进行以下设置**：

| 设置项 | 操作路径 |
|--------|---------|
| 🔋 关闭电池优化 | 设置 → 电池 → 不受限的应用 → 找到 V免签 → 开启 |
| 🔒 锁定后台 | 最近任务列表 → V免签卡片 → 点击锁定图标 |
| 🚀 允许自启动 | 设置 → 应用 → 自启动管理 → V免签 → 开启 |
| 🔔 确认通知开启 | 确保微信、支付宝的通知权限已开启 |

### 第五步：实际测试

1. 打开微信或支付宝的 **收款码**
2. 用另一台手机/账号扫码支付一笔小额（如 0.01 元）
3. 检查 V免签 服务端后台是否收到回调
4. 完成！系统正式运行 ✅

## 🔍 工作原理详解

### 核心监听机制

本 App 使用 Android 系统级 API `NotificationListenerService` 监听所有通知栏消息：

```
手机收到任意通知 → onNotificationPosted() 被系统回调
     ↓
判断通知来源包名
  ├─ com.tencent.mm (微信) → 检查标题/内容关键词 → 提取金额 → 推送
  ├─ com.eg.android.AlipayGphone (支付宝) → 检查标题/内容关键词 → 提取金额 → 推送
  └─ 其他 App → 仅记录日志
```

### 识别规则

**微信收款识别：**
- 标题必须包含：`微信收款助手`、`微信支付`、`微信收款商业版` 等
- 内容必须包含：`收款`、`到账`、`元` 等
- 双重校验，避免普通聊天消息误触发

**支付宝收款识别：**
- 标题或内容包含：`通过扫码向你付款`、`成功收款`、`收钱码收款通知` 等
- 二次验证内容中是否含有金额相关词

**金额提取（四级正则匹配）：**
1. `¥100.50` → 优先匹配货币符号
2. `收款100.50元` → 匹配关键词+金额
3. `100.50元` → 匹配纯数字+元
4. 兜底方案 → 提取合理数字

### 保活策略

| 策略 | 原理 | 效果 |
|------|------|------|
| 前台服务 | `startForeground()` 显示持久通知 | ⭐⭐⭐ 系统最高优先级 |
| WakeLock | 持有 CPU 电源锁 | ⭐⭐ 防止深度休眠 |
| 心跳线程 | 每 30 秒与服务端通信 | ⭐⭐ 保持进程活跃 |
| 自动重连 | `onListenerDisconnected` 时 `requestRebind()` | ⭐⭐ 断开后自恢复 |
| 开机自启 | `BootReceiver` 监听开机广播 | ⭐ 重启后自动运行 |

### 支持的收款场景

| 场景 | 微信 | 支付宝 |
|------|:----:|:------:|
| 个人收款码 | ✅ | ✅ |
| 商家二维码 | ✅ | ✅ |
| 收款商业版 | ✅ | — |
| 店员到账通知 | ✅ | ✅ |
| 转账到账 | ✅ | ✅ |

## 🛠️ 调试指南

### 查看日志

使用 ADB 连接手机后，运行以下命令查看实时日志：

```bash
# 查看所有 V免签 相关日志
adb logcat | grep "VmqNotification"

# 或者更精确地过滤
adb logcat -s VmqNotification:D MainActivity:D BootReceiver:D
```

### 日志含义速查表

| 日志关键词 | 含义 |
|-----------|------|
| `NotificationListener 已连接` | ✅ 监听服务启动成功 |
| `心跳成功: 200` | ✅ 与服务端通信正常 |
| `心跳失败` | ❌ 网络异常或服务端未启动 |
| `💬微信 新通知捕获` | 收到微信通知 |
| `💰支付宝 新通知捕获` | 收到支付宝通知 |
| `📦其他 新通知捕获` | 收到其他 App 通知（也会记录） |
| `[微信] ✅ 收款金额提取成功 ¥xx` | 微信收款识别成功 |
| `[支付宝] ✅ 收款金额提取成功 ¥xx` | 支付宝收款识别成功 |
| `推送响应: success` | 推送到服务端成功 |
| `⏭ 去重跳过` | 5 秒内重复通知被过滤 |
| `❌ 标题未匹配收款关键词` | 不是收款通知，已跳过 |

> 💡 **Tip:** 所有通知（包括非支付通知）都会在日志中打印完整信息，方便排查。

## ❓ 常见问题 FAQ

<details>
<summary><b>Q: 点击检测监听没有反应？</b></summary>

1. 进入手机 **设置 → 应用 → 特殊应用权限 → 通知使用权**，确认 V免签 已开启
2. 如果已开启，尝试 **关闭再重新开启**
3. 重启 V免签 App
4. 部分手机需要重启整机才能生效
</details>

<details>
<summary><b>Q: 微信/支付宝收款后没有回调？</b></summary>

1. 确认微信已关注 **"微信支付"** 公众号（否则收不到收款通知）
2. 确认支付宝的 **通知权限** 已开启
3. 确认 V免签 **未被电池优化杀死**（查看通知栏是否有"V免签监控运行中"的通知）
4. 用 `adb logcat | grep VmqNotification` 查看日志，确认是否收到通知
</details>

<details>
<summary><b>Q: 心跳检测失败？</b></summary>

1. 确认服务端已启动并可以正常访问
2. 确认配置的地址和密钥正确
3. 如果服务端使用 HTTPS，请在代码中修改 URL 前缀
4. 检查手机网络是否正常
</details>

<details>
<summary><b>Q: 服务经常被杀死？</b></summary>

请务必完成以下所有步骤：
1. 关闭电池优化（设置 → 电池 → V免签 → 不受限制）
2. 锁定后台（最近任务 → 锁定 V免签）
3. 开启自启动权限
4. 参考下方 **各品牌特殊设置**
</details>

<details>
<summary><b>Q: 金额识别错误怎么办？</b></summary>

1. 用 `adb logcat | grep "金额提取"` 查看提取过程
2. 日志会显示通知原文和每一级正则的匹配结果
3. 如果是新格式的通知，可能需要在 `NeNotificationService2.java` 中更新关键词或正则
</details>

<details>
<summary><b>Q: 支持哪些 Android 版本？</b></summary>

- 最低支持：Android 7.0 (API 24)
- 推荐运行：Android 12 ~ 15
- 已适配：Android 14/15 的前台服务类型声明、通知权限等新特性
</details>

## 📱 各品牌手机特殊设置

不同品牌的 Android 手机对后台应用有不同的限制策略，**请务必按照你的手机品牌进行设置**：

### 小米 / Redmi (MIUI / HyperOS)
```
设置 → 应用设置 → 授权管理 → 自启动管理 → 启用 V免签
设置 → 电池与性能 → V免签 → 无限制
最近任务 → V免签 → 点击锁定图标
```

### 华为 / 荣耀 (HarmonyOS / EMUI)
```
设置 → 电池 → 应用启动管理 → V免签 → 手动管理
  ✓ 允许自启动  ✓ 允许后台活动  ✓ 允许关联启动
设置 → 应用 → 应用管理 → V免签 → 耗电详情 → 后台高耗电允许
```

### OPPO / 一加 (ColorOS)
```
设置 → 电池 → 应用耗电管理 → V免签 → 允许后台运行
设置 → 应用 → 自启动管理 → 允许 V免签
手机管家 → 权限隐私 → 自启动管理 → 允许 V免签
```

### vivo / iQOO (OriginOS)
```
i管家 → 应用管理 → 自启动 → 启用 V免签
设置 → 电池 → 后台高耗电 → 允许 V免签
```

### 三星 (OneUI)
```
设置 → 电池 → 后台使用限制 → 将 V免签 移出限制列表
设置 → 应用 → V免签 → 电池 → 不受限
```
