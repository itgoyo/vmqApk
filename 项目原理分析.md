# V免签 Android监控端 项目原理分析

## 项目概述

V免签是一个**个人收款监控系统**，通过Android手机监听微信和支付宝的收款通知，自动将收款信息回调到服务端，实现免签约的个人收款解决方案。

---

## 核心技术原理

### 一、如何监听支付内容 📱

#### 1. **使用 NotificationListenerService 监听通知**

核心类：`NeNotificationService2` 继承自 `NotificationListenerService`

```
NotificationListenerService 是 Android 系统提供的服务，允许应用监听系统所有的通知栏消息
```

**关键实现：**

- **服务注册**（AndroidManifest.xml）：
```xml
<service
    android:name="com.vone.vmq.NeNotificationService2"
    android:permission="android.permission.BIND_NOTIFICATION_LISTENER_SERVICE">
    <intent-filter>
        <action android:name="android.service.notification.NotificationListenerService" />
    </intent-filter>
</service>
```

- **权限申请**：需要用户手动在系统设置中授予"通知使用权"
  - MainActivity 启动时检测权限：`isNotificationListenersEnabled()`
  - 如果未授权，自动跳转到系统设置页面：`gotoNotificationAccessSetting()`

#### 2. **监听回调方法**

```java
@Override
public void onNotificationPosted(StatusBarNotification sbn) {
    // 当系统收到任何通知时，此方法被回调
    String pkg = sbn.getPackageName();  // 获取通知来源应用包名
    Notification notification = sbn.getNotification();
    Bundle extras = notification.extras;
    
    // 提取通知内容
    String title = extras.getString(NotificationCompat.EXTRA_TITLE, "");
    String content = extras.getString(NotificationCompat.EXTRA_TEXT, "");
}
```

#### 3. **识别支付通知**

通过**包名**和**关键词匹配**识别微信/支付宝的收款通知：

**支付宝识别逻辑：**
```java
if (pkg.equals("com.eg.android.AlipayGphone")) {
    // 支付宝包名
    if (content.indexOf("通过扫码向你付款")!=-1 || content.indexOf("成功收款")!=-1) {
        // 匹配收款关键词
        String money = getMoney(content);  // 提取金额
        appPush(2, Double.valueOf(money)); // 推送到服务端，type=2表示支付宝
    }
}
```

**微信识别逻辑：**
```java
if (pkg.equals("com.tencent.mm")) {
    // 微信包名
    if (title.equals("微信支付") || title.equals("微信收款助手") || title.equals("微信收款商业版")) {
        // 匹配收款通知标题
        String money = getMoney(content);  // 提取金额
        appPush(1, Double.valueOf(money)); // type=1表示微信
    }
}
```

#### 4. **金额提取算法**

```java
public static String getMoney(String content) {
    List<String> ss = new ArrayList<String>();
    // 使用正则表达式，将所有非数字和小数点的字符替换为逗号
    for(String sss : content.replaceAll("[^0-9.]", ",").split(",")) {
        if (sss.length() > 0)
            ss.add(sss);
    }
    // 返回最后一个数字（通常是金额）
    return ss.size() < 1 ? null : ss.get(ss.size()-1);
}
```

**示例：**
- 通知内容："张三通过扫码向你付款100.50元"
- 提取结果："100.50"

### 二、如何保活App 🔋

V免签使用了**多种保活策略**确保监听服务不被系统杀死：

#### 1. **WakeLock 电源锁 - 防止CPU休眠**

```java
@SuppressLint("InvalidWakeLockTag")
public void acquireWakeLock(Context context) {
    PowerManager pm = (PowerManager)context.getSystemService(Context.POWER_SERVICE);
    // PARTIAL_WAKE_LOCK：保持CPU运行，屏幕和键盘可以关闭
    mWakeLock = pm.newWakeLock(
        PowerManager.PARTIAL_WAKE_LOCK | PowerManager.ON_AFTER_RELEASE, 
        "WakeLock"
    );
    mWakeLock.acquire();  // 获取锁
}
```

**作用：** 即使手机息屏，CPU 仍保持运行，后台线程继续工作

**权限：** `<uses-permission android:name="android.permission.WAKE_LOCK"/>`

#### 2. **心跳线程 - 保持服务活跃**

```java
public void initAppHeart() {
    newThread = new Thread(new Runnable() {
        @Override
        public void run() {
            while (true) {
                // 每30秒向服务端发送心跳请求
                String t = String.valueOf(new Date().getTime());
                String sign = md5(t + key);
                
                OkHttpClient okHttpClient = new OkHttpClient();
                Request request = new Request.Builder()
                    .url("http://" + host + "/appHeart?t=" + t + "&sign=" + sign)
                    .build();
                
                call.enqueue(callback);
                
                Thread.sleep(30 * 1000);  // 休眠30秒
            }
        }
    });
    newThread.start();
}
```

**作用：**
- 定期与服务端通信，保持网络活跃状态
- 服务端可监控监听端是否在线
- 避免系统因长时间无活动而回收进程

#### 3. **NotificationListenerService 系统级服务**

```java
@Override
public void onListenerConnected() {
    // 当监听服务连接成功时启动心跳
    initAppHeart();
}
```

**优势：**
- `NotificationListenerService` 是**系统级服务**，优先级较高
- 系统会尽量保持此类服务存活
- 即使被杀死，系统也会尝试重启

#### 4. **服务重启机制**

```java
private void toggleNotificationListenerService(Context context) {
    PackageManager pm = context.getPackageManager();
    
    // 先禁用服务
    pm.setComponentEnabledSetting(
        new ComponentName(context, NeNotificationService2.class),
        PackageManager.COMPONENT_ENABLED_STATE_DISABLED, 
        PackageManager.DONT_KILL_APP
    );
    
    // 再启用服务（强制重启）
    pm.setComponentEnabledSetting(
        new ComponentName(context, NeNotificationService2.class),
        PackageManager.COMPONENT_ENABLED_STATE_ENABLED, 
        PackageManager.DONT_KILL_APP
    );
}
```

**在 MainActivity 启动时调用**，确保服务处于激活状态

## 核心代码解析

### 1. 数据推送到服务端

```java
public void appPush(int type, double price) {
    String t = String.valueOf(new Date().getTime());
    
    // 签名算法：MD5(type + price + timestamp + key)
    String sign = md5(type + "" + price + t + key);
    
    String url = "http://" + host + "/appPush?t=" + t 
                 + "&type=" + type 
                 + "&price=" + price 
                 + "&sign=" + sign;
    
    OkHttpClient okHttpClient = new OkHttpClient();
    Request request = new Request.Builder().url(url).build();
    call.enqueue(callback);
}
```

**参数说明：**
- `type`: 1=微信，2=支付宝
- `price`: 收款金额
- `t`: 时间戳
- `sign`: MD5签名，防止篡改

### 2. MD5 签名算法

```java
public static String md5(String string) {
    MessageDigest md5 = MessageDigest.getInstance("MD5");
    byte[] bytes = md5.digest(string.getBytes());
    String result = "";
    for (byte b : bytes) {
        String temp = Integer.toHexString(b & 0xff);
        if (temp.length() == 1) {
            temp = "0" + temp;
        }
        result += temp;
    }
    return result;
}
```

**作用：** 确保通信安全，服务端验证请求合法性

## 保活策略总结

| 策略 | 实现方式 | 作用 |
|------|---------|------|
| **电源锁** | WakeLock | 防止CPU休眠 |
| **心跳线程** | 30秒循环请求 | 保持进程活跃 |
| **系统服务** | NotificationListenerService | 高优先级服务 |
| **服务重启** | 禁用→启用 | MainActivity启动时重启服务 |
| **白名单提示** | 用户手动操作 | 避免被系统清理 |

## 局限性与注意事项

1. **手机兼容性问题**
   - 不同厂商的ROM对后台服务限制不同（小米、华为等）
   - 需要手动添加到内存清理白名单
   - 部分手机无法正常监听通知

2. **依赖通知推送**
   - 必须保持微信/支付宝后台运行
   - 需要关注"微信支付"公众号
   - 关闭通知将无法监听

3. **不适合商用**
   - 只能监听一个账号的收款
   - 需要专门的手机挂机
   - 存在延迟（依赖通知推送速度）

4. **安全性**
   - MD5已被认为不够安全（建议升级为SHA256）
   - 需要确保服务端地址安全（建议使用HTTPS）

## 总结

V免签通过巧妙利用 Android 的 **NotificationListenerService** 实现了对微信和支付宝收款通知的实时监听，配合 **WakeLock 电源锁** 和 **心跳线程** 等保活策略，确保监听服务在后台持续运行。这是一个典型的"曲线救国"方案，在没有官方支付API的情况下，通过监听通知栏消息实现了收款监控功能。

**核心创新点：**
- 使用系统级服务（NotificationListenerService）提高进程优先级
- 多重保活策略确保服务稳定运行
- 通过正则表达式智能提取金额信息
- MD5签名保证数据传输安全性

这个项目非常适合个人开发者学习 Android 服务保活、通知监听等技术，但不建议用于商业环境。
